1. 整体架构概览

整个工作流共 10 个节点，流程如下：

核心逻辑：

阶段 说明
视频处理 上传视频 → 调用 FFmpeg 服务提取关键帧 → 用 AI 生成分镜描述
格式化 将 AI 返回结果转为 Markdown 格式 + 飞书表格格式
飞书同步 获取飞书 Token → 创建多维表格记录 → 同步分镜截图

2. 前置准备

在开始搭建之前，你需要准备以下内容：

2.1 部署 FFmpeg 分镜服务

你需要一个自部署的 HTTP 服务（ffmpeg-server:5002），提供以下 3 个 API：

接口 方法 说明
/extract POST 接收 {"file_url": "视频URL"}，返回关键帧截图及时间信息
/describe POST 接收 shots 数据，返回每帧的 AI 分镜描述
/feishu-create POST 将分镜数据写入飞书多维表格
/feishu POST 将分镜截图同步上传到飞书附件字段

提示： 如果你还没有这个服务，需要自己搭建或使用已有的 FFmpeg + AI 描述服务。

2.2 飞书开放平台配置

  1. 登录 飞书开放平台 创建一个企业自建应用
  2. 获取 App ID 和 App Secret
  3. 创建一个多维表格，记录 app_token 和 table_id
  4. 在多维表格中创建以下字段：
字段名 类型 说明
分镜编号 文本 如 “分镜01”
时间范围 文本 如 “0-5”
画面主体 文本 画面中的主要对象
动作 文本 画面中的动作描述
环境 文本 场景环境描述
运镜 文本 镜头运动方式
AI描述 文本 AI 生成的完整描述
  1. 给应用添加权限：bitable:app（多维表格读写）

2.3 Dify 环境

  • 已部署 Dify（版本 ≥ 0.6.0）
  • 有创建工作流应用的权限

3. 步骤一：创建 Dify 工作流应用

  1. 打开 Dify 后台，点击 「创建应用」
  2. 选择应用类型为 「工作流编排」
  3. 应用名称填写：视频反推分镜Agent
  4. 图标选择 🎬，背景色设置为 #FFEAD5
  5. 描述填写：上传视频，自动拆分分镜，生成分镜脚本并写入飞书多维表格

13. 运行测试

13.1 发布应用

  1. 点击右上角的 「发布」 按钮
  2. 确认所有节点配置无误

13.2 测试运行

  1. 在应用的运行界面，上传一个测试视频（推荐使用 30 秒以内的短视频）
  2. 等待工作流执行完成
  3. 预期输出：
    • 📄 一份格式化的 Markdown 分镜脚本（包含每个分镜的时间范围、画面主体、动作、环境、运镜等）
    • 🔢 分镜总数
    • 📊 飞书多维表格中自动新增对应行

13.3 常见问题排查

问题 可能原因 解决方案
提取关键帧超时 视频文件过大或 FFmpeg 服务响应慢 增加超时时间，或压缩视频后重试
飞书写入失败 App ID/Secret 错误或权限不足 检查飞书应用权限，确认凭证正确
图片同步失败 img_field_id 不匹配 在飞书多维表格中确认图片字段的 ID
描述为空 FFmpeg 服务的 /describe 接口异常 检查 FFmpeg 服务日志

附录：一键导入

如果你想直接导入而不是手动搭建，可以将原始的 .yml 文件通过 Dify 的 「导入DSL」 功能一键导入：

  1. 打开 Dify → 创建应用 → 选择 「导入DSL文件」
  2. 上传 视频反推分镜Agent (6).yml
  3. 导入后，根据本教程替换飞书凭证等环境变量即可使用
