从零开始，搭建基于RAG的平安保险智能客服问答系统

本教程将带你从零开始，搭建一个基于 RAG（检索增强生成）的保险产品智能问答系统。
系统使用阿里云百炼大模型 + LangGraph Agent，支持 7 种平安保险产品的知识问答、报告生成和邮件通知。

1. 项目简介

本项目是一个 保险产品智能问答 Agent，核心能力包括：

能力 说明
🔍 知识检索 从 7 种平安保险产品文档中精准检索相关内容
💬 智能问答 基于 ReAct（推理+行动）模式的多轮对话
📊 报告生成 自动生成结构化的保险产品分析报告
💾 报告保存 将报告保存为本地文件
📧 邮件通知 模拟邮件发送（预留 SMTP 集成接口）

支持的保险产品：

  1. 平安商业综合责任保险（亚马逊）
  2. 雇主责任险
  3. 平安企业团体综合意外险
  4. 雇主安心保
  5. 施工保
  6. 财产一切险
  7. 平安装修保

2. 技术架构总览

数据流向：

  1. docs/*.txt → 索引器 → 分块 → 向量化 → 存储到 data/
  2. 用户提问 → Embedding → 余弦相似度检索 → 取 Top 5 相关片段
  3. 相关片段 + 用户问题 → LLM（通义千问） → 生成回答
  4. 复杂任务时 Agent 自动调用工具链：检索 → 生成报告 → 保存 → 发送邮件

3.2 获取阿里云百炼 API Key

本项目使用阿里云百炼（DashScope）平台的大模型服务，需要一个 API Key。

步骤：

  1. 访问 阿里云百炼控制台
  2. 注册/登录阿里云账号
  3. 开通百炼服务（首次使用需要开通）
  4. 进入 API-KEY 管理 页面
  5. 点击 创建新的 API-KEY
  6. 复制生成的 Key（格式类似 sk-xxxxxxxxxxxxxxxx）

⚠️ 注意：API Key 请妥善保管，不要泄露到公开仓库中。
使用通义千问模型可能产生费用，请关注阿里云的计费说明和免费额度。

11. 核心模块详解

11.1 配置模块 — src/config.mjs

集中管理所有配置项，通过 .env 文件和环境变量覆盖默认值：

配置项 默认值 说明
DASHSCOPE_API_KEY （必填） 阿里云百炼 API Key
MODEL_NAME qwen3.6-max-preview 聊天模型名称
EMBEDDING_MODEL text-embedding-v2 Embedding 模型名称
baseUrl https://dashscope.aliyuncs.com/compatible-mode/v1 API 基础地址
port 3000 服务端口
chunkSize 500 文本分块大小（字符数）
chunkOverlap 100 分块重叠大小（字符数）
retrievalTopK 5 检索返回的最相关片段数

11.2 文档索引器 — src/indexer.mjs

索引器负责将原始文档转化为可检索的向量数据，流程如下：

关键设计：

  • 使用 RecursiveCharacterTextSplitter 进行智能分块，保持语义完整性
  • 每 20 个片段一批调用 Embedding API，避免请求过大
  • 向量维度：1536（text-embedding-v2 模型输出）

11.3 向量检索引擎 — src/retriever.mjs

使用纯 JavaScript 实现的余弦相似度检索：

工作流程：

  1. 将用户问题转为 1536 维向量
  2. 与所有 54 个文档向量计算余弦相似度
  3. 按相似度排序，返回 Top 5 最相关片段

💡 当前使用线性扫描（遍历全部向量），对于 54 个片段在毫秒内完成。
如果文档量增长到数千条以上，建议引入向量数据库（如 Milvus、Pinecone）。

11.4 Agent 工具 — src/tools.mjs

系统为 Agent 定义了 4 个工具，Agent 根据用户意图自动选择调用：

工具名 功能 使用场景
search_knowledge_base 检索知识库 用户询问保险产品信息时
generate_report 生成分析报告 用户需要综合分析报告时
save_report 保存报告到文件 用户要求保存报告时
send_email 发送邮件（模拟） 用户要求邮件通知时

11.6 HTTP 服务器 — src/server.mjs

基于 Express.js 构建的 Web 服务器：

  • 静态文件服务：提供 public/index.html 前端页面
  • 会话管理：使用内存 Map 存储会话历史，每个会话最多保留 40 条消息（20 轮对话）
  • API 路由：提供健康检查、索引构建、问答、会话清除 4 个接口

⚠️ 当前会话存储在内存中，服务重启后会话将丢失。
生产环境建议接入 Redis 进行会话持久化。

13. 常见问题排查

❌ Error: DASHSCOPE_API_KEY is required

原因：未配置 API Key 或 .env 文件路径错误。

解决：

  • 确认 node-qa-agent/.env 文件存在
  • 确认文件中包含 DASHSCOPE_API_KEY=sk-xxxxxx
  • 确认 Key 值没有多余的空格或引号

❌ 索引构建失败 或 Embedding API 调用失败

原因：API Key 无效、余额不足或网络问题。

解决：

  1. 登录百炼控制台确认 API Key 有效
  2. 检查账户余额是否充足
  3. 确认网络能访问 dashscope.aliyuncs.com
  4. 如果在中国大陆以外地区，可能需要配置代理

❌ 找不到模块 或 import 错误

原因：依赖未正确安装。

解决：

❌ AI 回答质量不佳 / 回答与文档不符

可能原因：

  1. 索引未更新（修改了文档但未重建索引）
  2. 文档格式问题（编码、乱码）
  3. 分块参数不合理

解决：

  1. 重新执行 npm run index
  2. 确认文档为 UTF-8 编码的纯文本格式
  3. 尝试调整 chunkSize 和 chunkOverlap

❌ 端口 3000 已被占用

解决：

❌ ECONNREFUSED 连接被拒绝

原因：服务未启动或地址不正确。

解决：

  1. 确认服务正在运行（终端中应有”服务已启动”的提示）
  2. 确认访问地址为 http://localhost:3000
  3. 检查防火墙是否阻止了 3000 端口
