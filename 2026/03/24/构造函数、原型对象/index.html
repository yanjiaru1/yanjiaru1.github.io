二、改变 this 指向的三种方法

1. call、apply、bind 对比

方法 执行时机 传参方式 返回值 使用场景
call 立即执行 fn.call(obj, arg1, arg2) 函数执行结果 参数明确，个数固定
apply 立即执行 fn.apply(obj, [arg1, arg2]) 函数执行结果 参数是数组形式
bind 返回新函数 fn.bind(obj, arg1, arg2) 绑定 this 的新函数 需要延迟执行

四、IIFE（立即执行函数表达式）详解

语法

作用

作用 说明
作用域隔离 内部变量不会污染全局
模块化封装 隐藏内部实现细节
避免命名冲突 多个插件可以共存
闭包保存状态 内部变量常驻内存

为什么插件要用 IIFE？

  • 防止全局变量污染
  • 保护内部实现不被外部修改
  • 只暴露必要的 API（window.myjQuery）

五、插件开发的核心设计模式

六、知识点总结表

分类 核心概念 关键要点
构造函数 new 操作符 4步流程：创建对象 → 绑定 this → 执行 → 返回
原型对象 prototype、__proto__、constructor 三者相互关联，实现属性和方法共享
this 指向 call / apply / bind call/apply 立即执行，bind 返回新函数
闭包 IIFE 隔离作用域，避免全局污染
插件模式 构造函数 + 原型 + 工厂函数 封装 DOM 操作，暴露统一 API
链式调用 return this 方法返回实例本身，支持连续调用

七、改进建议（支持链式调用和多元素）

八、核心要点回顾

  1. 构造函数 + new：创建对象的模板
  2. 原型对象：共享方法，节省内存
  3. call/apply/bind：动态改变 this 指向
  4. IIFE：隔离作用域，封装插件
  5. 工厂函数：暴露简洁 API
  6. 链式调用：return this 实现

这套模式是 jQuery 和众多 JavaScript 库的经典实现方式！
